# Lanchu — Data model (SQLite)

> The v0 data model. This makes [`ARCHITECTURE.md`](./ARCHITECTURE.md) concrete in terms of tables,
> fields, and states. Standard SQLite (compatible with `node:sqlite`); JSON stored as `TEXT` with
> the `json1` functions. All timestamps are ISO-8601 UTC (`TEXT`). The `id`s
> are strings generated by the app except where noted.

---

## 1. Overview (entities and relationships)

```
org ─┬─ role ──< role_tag
     ├─ agent ──< session
     │     └── (owner) ──< task
     ├─ project ──< task ──< task_tag
     │                └──< task_dep
     ├─ doc
     └─ event   (append-only: audit log + event stream)
```

- **org** contains everything else. There can be several on the same machine.
- **role** = name + allowed tags (via `role_tag`) or wildcard.
- **agent** = durable member (active/idle/retired), with a role.
- **session** = live connection tied to an agent (ephemeral; identity by token).
- **project** groups **task**s; each task carries **task_tag**s and **task_dep**endencies.
- **doc** = shared org documentation.
- **event** = immutable record; it is both the audit log and the source of the stream.

## 2. DDL

```sql
PRAGMA journal_mode = WAL;      -- concurrent reads; a single writer (the server)
PRAGMA foreign_keys = ON;

CREATE TABLE schema_meta (
  version     INTEGER NOT NULL
);
-- INSERT INTO schema_meta(version) VALUES (1);

CREATE TABLE org (
  id          TEXT PRIMARY KEY,
  name        TEXT NOT NULL UNIQUE,
  created_at  TEXT NOT NULL
);

CREATE TABLE project (
  id          TEXT PRIMARY KEY,
  org_id      TEXT NOT NULL REFERENCES org(id) ON DELETE CASCADE,
  name        TEXT NOT NULL,
  created_at  TEXT NOT NULL,
  UNIQUE (org_id, name)
);

CREATE TABLE role (
  id          TEXT PRIMARY KEY,
  org_id      TEXT NOT NULL REFERENCES org(id) ON DELETE CASCADE,
  name        TEXT NOT NULL,
  is_wildcard INTEGER NOT NULL DEFAULT 0,   -- 1 = can touch any tag ('*')
  created_at  TEXT NOT NULL,
  UNIQUE (org_id, name)
);

CREATE TABLE role_tag (
  role_id     TEXT NOT NULL REFERENCES role(id) ON DELETE CASCADE,
  tag         TEXT NOT NULL,
  PRIMARY KEY (role_id, tag)
);

CREATE TABLE agent (
  id                TEXT PRIMARY KEY,
  org_id            TEXT NOT NULL REFERENCES org(id) ON DELETE CASCADE,
  role_id           TEXT NOT NULL REFERENCES role(id),
  name              TEXT NOT NULL,            -- display, unique per org (e.g. 'fix-login')
  objective         TEXT,                     -- objective it was created with (for reuse + panel)
  state             TEXT NOT NULL DEFAULT 'active'
                      CHECK (state IN ('active','idle','retired')),
  last_activity_at  TEXT,                     -- derived from the last tool-call (panel)
  last_activity     TEXT,                     -- short summary of that action
  created_at        TEXT NOT NULL,
  retired_at        TEXT,
  UNIQUE (org_id, name)
);

CREATE TABLE session (
  id          TEXT PRIMARY KEY,
  agent_id    TEXT NOT NULL REFERENCES agent(id) ON DELETE CASCADE,
  token       TEXT NOT NULL UNIQUE,           -- issued by the launcher; identifies the connection
  client      TEXT,                           -- e.g. 'claude-code', 'cursor'
  started_at  TEXT NOT NULL,
  ended_at    TEXT                            -- NULL = live → agent ACTIVE
);

CREATE TABLE task (
  id                  TEXT PRIMARY KEY,        -- e.g. 'task-42'
  project_id          TEXT NOT NULL REFERENCES project(id) ON DELETE CASCADE,
  parent_task_id      TEXT REFERENCES task(id),-- NULL = root (usually the objective)
  title               TEXT NOT NULL,
  status              TEXT NOT NULL DEFAULT 'available'
                        CHECK (status IN ('available','claimed','in_progress','blocked','done')),
  owner_agent_id      TEXT REFERENCES agent(id),-- NULL iff status='available'
  workspace           TEXT,                    -- generic: git branch, folder, board… (optional)
  created_by_agent_id TEXT REFERENCES agent(id),
  created_at          TEXT NOT NULL,
  claimed_at          TEXT,                    -- to detect 'stale' (idle zombie)
  updated_at          TEXT NOT NULL,
  done_at             TEXT
);

CREATE TABLE task_tag (
  task_id     TEXT NOT NULL REFERENCES task(id) ON DELETE CASCADE,
  tag         TEXT NOT NULL,
  PRIMARY KEY (task_id, tag)
);

CREATE TABLE task_dep (
  task_id            TEXT NOT NULL REFERENCES task(id) ON DELETE CASCADE,
  depends_on_task_id TEXT NOT NULL REFERENCES task(id) ON DELETE CASCADE,
  PRIMARY KEY (task_id, depends_on_task_id)
);

CREATE TABLE doc (
  id                  TEXT PRIMARY KEY,
  org_id              TEXT NOT NULL REFERENCES org(id) ON DELETE CASCADE,
  title               TEXT NOT NULL,
  content             TEXT NOT NULL DEFAULT '',
  updated_at          TEXT NOT NULL,
  updated_by_agent_id TEXT REFERENCES agent(id),
  created_at          TEXT NOT NULL
);

-- Append-only. It is the audit log and the source of the event stream.
CREATE TABLE event (
  id             INTEGER PRIMARY KEY AUTOINCREMENT,  -- monotonic stream order
  org_id         TEXT NOT NULL REFERENCES org(id) ON DELETE CASCADE,
  project_id     TEXT REFERENCES project(id) ON DELETE SET NULL,
  type           TEXT NOT NULL,                       -- see §3
  actor_agent_id TEXT REFERENCES agent(id),
  subject_kind   TEXT,                                -- 'task' | 'doc' | 'agent'
  subject_id     TEXT,
  workspace      TEXT,
  tokens         INTEGER,                             -- self-reported; may be NULL
  outcome        TEXT NOT NULL DEFAULT 'applied'
                   CHECK (outcome IN ('applied','rejected')),  -- 'rejected' = scope.violation
  data           TEXT,                                -- extra JSON
  created_at     TEXT NOT NULL
);

CREATE INDEX idx_task_project_status ON task(project_id, status);
CREATE INDEX idx_task_owner          ON task(owner_agent_id);
CREATE INDEX idx_task_tag_tag        ON task_tag(tag);
CREATE INDEX idx_role_tag_tag        ON role_tag(tag);
CREATE INDEX idx_agent_org_state     ON agent(org_id, state);
CREATE INDEX idx_session_agent_live  ON session(agent_id, ended_at);
CREATE INDEX idx_event_org_id        ON event(org_id, id);
CREATE INDEX idx_event_actor         ON event(actor_agent_id, id);
```

## 3. Enumerations

- **agent.state:** `active` · `idle` · `retired`.
- **task.status:** `available` · `claimed` · `in_progress` · `blocked` · `done`.
  Invariant: `status='available'` ⟺ `owner_agent_id IS NULL`.
  "Reserved" (idle) is **not** a status: it's a `claimed`/`in_progress` task whose owner
  is `idle`.
- **event.type:** `agent.created` · `agent.reused` · `agent.active` · `agent.idle` ·
  `agent.retired` · `task.created` · `task.claimed` · `task.released` · `task.started` ·
  `task.completed` · `task.blocked` · `task.reassigned` · `task.handoff` ·
  `doc.created` · `doc.updated` · `scope.violation`.
- **event.outcome:** `applied` (it was applied) · `rejected` (it was blocked; e.g. `scope.violation`).

## 4. Key operations (how the schema makes each decision real)

### 4.1 Atomic claim + scope check (A2)
Inside a `BEGIN IMMEDIATE` transaction (a single writer with WAL):
```sql
-- 1) Does the agent's role cover ALL of the task's tags?  (or is it a wildcard)
--    Reject if any task_tag exists outside the role's allowed tags.
SELECT 1
FROM task t
JOIN agent a   ON a.id = :agent_id
JOIN role  r   ON r.id = a.role_id
WHERE t.id = :task_id
  AND ( r.is_wildcard = 1
        OR NOT EXISTS (
          SELECT 1 FROM task_tag tt
          WHERE tt.task_id = t.id
            AND tt.tag NOT IN (SELECT tag FROM role_tag WHERE role_id = r.id)
        ) );
-- if it returns no row → REJECT: emit event(type='scope.violation', outcome='rejected')

-- 2) Atomic lock: only succeeds if it's still available.
UPDATE task
SET status='claimed', owner_agent_id=:agent_id, claimed_at=:now, updated_at=:now
WHERE id=:task_id AND status='available';
-- changes() must be 1; if it's 0 → it was already taken (someone else won the race).
```
`task.create` applies the **same** scope check from step 1 over the `tags`
being set.

### 4.2 Presence and activity (A4)
- **Active:** `EXISTS (SELECT 1 FROM session WHERE agent_id=? AND ended_at IS NULL)`.
- **Idle:** the agent has no live session and is not retired.
- **Activity ("what it's doing now"):** every state-mutating tool-call updates
  `agent.last_activity_at` / `last_activity` and writes an `event`. The panel reads that; there's
  no heartbeat from the agent.

### 4.3 Reuse by objective (A1 / footprint)
Given a new objective, find `idle` agents in the org whose **footprint** overlaps:
```sql
-- Footprint = agent.objective + the titles/tags of its tasks.
-- v0: keyword/tag overlap, ranked by number of matches.
-- (Fine-grained matching is left as an implementation detail; see OPEN-QUESTIONS.)
```
The candidates are offered ranked; the choice is made by the human in the **CLI**.

### 4.4 Dependency unblocking
When a task moves to `done`, review the ones that depended on it:
```sql
-- a 'blocked' task can go back to 'available' if it has NO remaining deps that aren't 'done'
UPDATE task SET status='available', updated_at=:now
WHERE status='blocked'
  AND NOT EXISTS (
    SELECT 1 FROM task_dep d JOIN task p ON p.id = d.depends_on_task_id
    WHERE d.task_id = task.id AND p.status <> 'done' );
```

### 4.5 Safe retirement (handoff)
```sql
-- BLOCKS if the agent has open tasks:
SELECT COUNT(*) FROM task
WHERE owner_agent_id=:agent_id AND status IN ('claimed','in_progress','blocked');
-- if > 0 → require for each one: task.reassign (owner→another) or task.release (→available)
-- only then:
UPDATE agent SET state='retired', retired_at=:now WHERE id=:agent_id;
UPDATE session SET ended_at=:now WHERE agent_id=:agent_id AND ended_at IS NULL;
```

## 5. Details this schema leaves ready (or flags)

- **Idle "zombie" (C4):** two **derived** signals (not new columns):
  - *reserved* = `owner_agent_id` is non-null and its `agent.state='idle'`.
  - *stale* = reserved **and** `updated_at` older than `LANCHU_STALE_HOURS` (default 24h).

  **v0: no auto-release.** The panel/`lanchu tasks` flag them; the supervisor uses
  `task release`/`task reassign` (override). Those overrides are audited with
  `event.data.override = true` (actor = supervisor, not the owner).
- **Multi-org/project:** the schema supports multiple orgs and projects. **How the
  launcher chooses the org/project and role for a new agent** is still open (C2/C3).
- **Docs always up to date (C5):** there's `doc` + traceability via `event`. The *mechanism*
  for "pushing to update" (a nudge on task close) is behavioral, not schema.
- **Cost/tokens:** `event.tokens` is **self-reported** and may be `NULL`.
- **Migrations:** `schema_meta.version` to version the schema across releases.
