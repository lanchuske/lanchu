What it is

Lanchu is a local-first, open-source control and trust layer for AI agents, built on the Model Context Protocol (MCP). It does not run or orchestrate your agents — it coordinates and bounds their work. Agents talk through Lanchu instead of directly, so nothing they do is a blind spot: work is claimed and mediated, roles limit what each agent can touch, and every action lands in an audit log you can watch live.

What it does

Durable agents

Agents persist as team members with a name, role and history — reuse them instead of spinning up duplicates.

Mediated coordination

Agents coordinate through Lanchu, never directly, so tasks are claimed without collisions and nothing is a blind spot.

Cooperative, auditable limits

Each role has a scope. Actions outside it are rejected and recorded — governance you can prove, not just hope for.

Real-time panel

See who is active, what they are working on, and a full history — a live supervisor view over the whole team.

How it works

  1. Start it in your project

    A guided wizard sets up your organization, agent and role.

    $ npx lanchu

  2. Connect your agent

    The wizard wires your MCP client (Claude Code) for you, or prints the command to run.

  3. Tell your agent to work in Lanchu

    It reads its objective, role and tasks, claims what is in its lane, and reports progress — all visible to you. Add more agents with different roles and they coordinate through Lanchu, never directly.

FAQ

What is Lanchu?

Lanchu is a local-first, open-source control and trust layer for AI coding agents. It coordinates multiple agents so they do not collide, keeps each one within a defined role, and gives you a real-time panel and a full audit history of everything they do.

Is Lanchu free and open source?

Yes. Lanchu is MIT-licensed and free to use. The source is on GitHub and it is published to npm as lanchu.

Does Lanchu run in the cloud?

No. Lanchu runs 100% on your machine. There is no public endpoint and no phone-home by design; your data never leaves your computer.

Which agents does it work with?

Any MCP-compatible agent. Lanchu is built around Claude Code and connects over the Model Context Protocol, so agents coordinate through Lanchu rather than directly.

Does Lanchu run or orchestrate my agents?

No. Lanchu does not execute or drive your agents. It coordinates and bounds their work: a shared workspace, roles and scope limits, and an audit trail. You keep using your own agents.

How do I install it?

Run npx lanchu. A guided wizard sets up your organization, agent and role, wires your MCP client and launches it. Lanchu requires Node.js 22.5 or newer.

What data does Lanchu store, and where?

Everything — organizations, agents, tasks, docs and the audit log — lives in a single local SQLite database in your operating system's app-data folder. Running lanchu uninstall --purge removes it completely.
